## 💡 CONTENU DÉTAILLÉ PAR SECTION

### L'Association El Awael
- Association marocaine à but non lucratif, région Tétouan
- Mission : éducation spécialisée et réhabilitation des enfants handicapés
- 3 centres : Martil (lieu du stage), M'diq, Fnideq (siège)
- Types de handicap : déficiences intellectuelles, TSA, handicaps moteurs, troubles sensoriels
- Outil pédagogique central : PEI (Projet Éducatif Individualisé) — document définissant les objectifs individuels de chaque enfant

### Situation avant intervention (AS-IS)
- 44 mots de passe en clair dans le fichier HTML source (risque sécurité critique)
- Données sur localStorage — perdues si changement de navigateur
- Aucun KPI ni tableau de bord — décision entièrement intuitive
- Registres de présences manuscrits
- PEI sur papier / fichiers Word
- Reporting direction : 3-4 heures par rapport mensuel
- Export limité à window.print()
- Interface avec alert() / confirm() natifs

### La plateforme développée
Technologies : Firebase (Firestore + Auth + Storage) + JavaScript pur + GitHub Pages — **coût total : 0 MAD/mois**

**Modules :**

**1. Authentification sécurisée**
- Firebase Auth avec tokens JWT
- 3 rôles : ADMIN / GESTIONNAIRE / ÉDUCATEUR / CONSULTANT
- Persistance de session (IndexedDB)
- Reset mot de passe par email

**2. Tableau de bord KPIs (8 indicateurs)**
- 🎯 Réalisation PEI
- 📅 Taux de présence
- 🚨 Bénéficiaires à risque élevé
- ⚖️ Ratio cadre/bénéficiaire
- 👨‍🏫 Cadres actifs ce mois
- 🔔 Alertes non résolues
- 📵 Sans séance ce mois
- 📋 Conformité dossiers

**3. Score Organisationnel Global (SOG)**
```
SOG = (Réalisation PEI × 40%) + (Présence × 30%) + (Indépendance × 20%) + (Dossiers × 10%)
≥70 = Excellent | ≥40 = Bien | <40 = À améliorer
```

**4. Score de risque individuel (0-100)**
```
+40 pts → présence < 50%
+25 pts → présence entre 50% et 70%
+35 pts → réalisation PEI < 30%
+20 pts → réalisation entre 30% et 50%
+15 pts → dossier incomplet
+10 pts → aucune séance ce mois
0-30 = Stable | 31-60 = Surveiller | 61-100 = Urgent
```

**5. Simulation stratégique**
- Leviers : présence (+0 à +30%), PEI (+0 à +30%), autonomie (+0 à +20%)
- Score simulé recalculé en temps réel
- Scénarios prédéfinis : Plan urgence / Effort équilibré / Viser Excellent

**6. Heatmap des risques**
- Vue par classe (vert/orange/rouge)
- Navigation du macro (classe) au micro (bénéficiaire)

**7. Journal d'audit**
- Toutes les actions tracées (connexion, CRUD, exports, suppressions)
- Horodatage + identification utilisateur
- Stocké dans Firestore collection audit_logs
